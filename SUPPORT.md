# Supporto

## Problemi con la web app

Se qualcosa non funziona — una storia non si salva, il PDF non si scarica, la stampa viene tagliata — apri una issue usando il template **Bug Report**.

Inserisci sempre: browser, versione del browser, sistema operativo, dispositivo, e i passi esatti per riprodurre il problema. Senza questi dati è difficile diagnosticare qualcosa.

## Richieste di funzionalità

Per proporre nuove funzionalità — tipi di frase aggiuntivi, nuovi esempi, opzioni di esport — usa il template **Feature Request**. Descrivi il caso d'uso educativo specifico: chi usa quella funzione, in quale contesto, con quale obiettivo. Le richieste legate a bisogni reali di insegnanti, terapisti o famiglie hanno priorità.

## Domande tecniche

Per domande sull'architettura o sul codice, consulta prima la [wiki tecnica](wiki/Home.md). Le domande già coperte dalla wiki potrebbero non ricevere risposta individuale.

## Tempi

Il progetto è mantenuto da una sola persona. I tempi di risposta variano. Le issue con contesto completo (browser, passi, screenshot) vengono gestite prima.
